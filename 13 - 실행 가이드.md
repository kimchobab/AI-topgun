## run_local_dogfight.py 기본 사용법

### 1. BT vs BT (기본 테스트)
```bash
python run_local_dogfight.py \
    --ownship-backend bt \
    --target-backend bt \
    --ownship-bt-dll AIP_DCS_ownship.dll \
    --target-bt-dll AIP_BASE_target.dll
```

### 2. RL vs BT (학습된 정책 평가)
```bash
python run_local_dogfight.py \
    --ownship-backend rl \
    --target-backend bt \
    --ownship-bundle-dir ./checkpoints/sac_bundle \
    --observation-mode tactical16 \
    --save-log
```

### 3. Hybrid vs BT (혼합 모드)
```bash
python run_local_dogfight.py \
    --ownship-backend hybrid \
    --target-backend bt \
    --ownship-bundle-dir ./checkpoints/sac_bundle \
    --hybrid-mode residual \
    --residual-scale 0.35
```

### 4. RL vs RL (자기대국)
```bash
python run_local_dogfight.py \
    --ownship-backend rl \
    --target-backend rl \
    --ownship-bundle-dir ./checkpoints/ownship_bundle \
    --target-bundle-dir ./checkpoints/target_bundle \
    --observation-mode tactical16
```

## 전체 CLI 옵션

| 옵션 | 기본값 | 설명 |
|------|--------|------|
| `--ownship-backend` | 필수 | rl / bt / hybrid / fixed |
| `--target-backend` | 필수 | rl / bt / hybrid / fixed |
| `--ownship-bundle-dir` | — | RL 번들 디렉토리 (rl/hybrid 필수) |
| `--target-bundle-dir` | — | 적기 RL 번들 디렉토리 |
| `--ownship-bt-dll` | AIP_DCS_ownship.dll | 아군 BT DLL |
| `--target-bt-dll` | AIP_BASE_target.dll | 적기 BT DLL |
| `--observation-mode` | tactical16 | classic12 / relative14 / tactical16 / custom |
| `--observation-module` | — | 커스텀 관측 모듈 이름 |
| `--hybrid-mode` | residual | residual / blend / switch |
| `--alpha` | 0.5 | blend 모드 가중치 |
| `--residual-scale` | 0.35 | residual 스케일 |
| `--max-engage-time` | 300.0 | 최대 교전 시간 (초) |
| `--episode-step-limit` | 18000 | 최대 스텝 수 |
| `--min-altitude` | 300.0 | 최소 고도 (m) |
| `--save-log` | False | TacView CSV 로그 저장 |
| `--bt-rule-xml` | — | BT 규칙 XML 경로 |

## 실행 전 체크리스트

> 처음 실행하거나 제출 전에 아래 5가지를 순서대로 확인한다.

- [ ] **1. observation_mode 3곳 일치** — `my_train.py` / `run_local_dogfight.py --observation-mode` / `my_submission.py` 모두 동일한 값 (권장: `tactical16`)
- [ ] **2. ACTION_REPEAT 일치** — `my_train.py`의 `step_ratio`(또는 `action_repeat`) 와 `my_submission.py`의 `ACTION_REPEAT = 6` 이 동일한가
- [ ] **3. 번들 경로 확인** — `--ownship-bundle-dir` 경로에 `metadata.json`과 `policy_weights.pkl` 이 모두 존재하는가
- [ ] **4. DLL 경로 확인** — BT/Hybrid 모드라면 `--ownship-bt-dll`, `--target-bt-dll` 경로가 실제 파일을 가리키는가
- [ ] **5. 작업 디렉토리 확인** — Release 폴더에서 실행 중인가 (`cd "...Release"`)

---

## ⚠️ 필수 체크

> ==훈련 시 step_ratio와 my_submission.py의 ACTION_REPEAT = 6 이 반드시 일치해야 함. 불일치 시 제출 후 전혀 다른 행동 발생.==

> ==훈련(my_train.py) → 로컬 테스트(run_local_dogfight.py) → 제출(my_submission.py) 세 곳의 observation_mode 가 반드시 동일해야 함.==

## 자주 발생하는 문제

| 문제 | 원인 | 해결 |
|------|------|------|
| `FDM Update Fail` | 물리 시뮬 발산 | 초기 속도/고도 조정 |
| `ownship altitude below min` | 급강하 | 저고도 패널티 강화 |
| `fuel fail` | 연료 소진 | max_engage_time 줄이기 |
| DLL not found | BT DLL 경로 오류 | 절대 경로로 지정 |
| CUDA OOM | GPU 메모리 부족 | batch_size 줄이기 |
