# 🚀 최소 성공 경로 (Quick Start)

[[00 - 전체 인덱스|← 인덱스로]]

> 처음 시작하는 학생을 위한 1페이지 최단 경로. 이 순서대로만 해도 제출 가능한 RL 에이전트를 만들 수 있다.

---

## Step 1 — observation_mode 고정

모든 파일에서 `tactical16`으로 통일한다.

```python
# my_train.py
ENV_CONFIG = {
    "observation_mode": "tactical16",
    ...
}
```

> `classic12`(코드 기본값)를 그대로 쓰면 WEZ·pursuit 피처가 없어 학습 효율이 떨어진다.

---

## Step 2 — 기본 보상으로 먼저 학습

`my_reward.py`를 수정하지 말고, 기본 `reward.py`를 그대로 연결해 일단 학습이 돌아가는지 확인한다.

```python
# my_train.py
TRAINING_CONFIG = {
    "reward_module": None,  # None = 기본 reward.py 사용
    ...
}
```

에이전트가 Stage 0(생존)과 Stage 1(접근)을 통과하면 기본 보상도 작동한다는 뜻이다.

---

## Step 3 — reward 파라미터 조절

기본 학습이 확인되면 `my_reward.py`에서 아래 3개만 먼저 조절한다.

| 파라미터 | 기본값 | 조절 방향 |
|----------|--------|----------|
| `step_penalty` | -0.001 | 더 키우면 빠른 교전 유도 |
| `pursuit_scale` | 0.3 | 키우면 WEZ 접근 강화 |
| `win_reward` | 100.0 | 키우면 승리 동기 강화 |

> 한 번에 여러 파라미터를 바꾸지 말 것. 변화 원인을 추적하기 어려워진다.

---

## Step 4 — 로컬 테스트

```bash
python run_local_dogfight.py \
    --ownship-backend rl \
    --target-backend bt \
    --ownship-bundle-dir ./artifacts/models/팀이름/v1 \
    --observation-mode tactical16 \
    --save-log
```

승률 50% 이상이면 제출 준비 완료.

---

## Step 5 — 제출 전 my_submission.py 동기화

아래 3가지가 `my_train.py`와 정확히 일치하는지 확인한다.

```python
# my_submission.py
OBSERVATION_MODE = "tactical16"      # ← my_train.py와 동일?
ACTION_REPEAT    = 6                 # ← my_train.py step_ratio와 동일?
BUNDLE_DIR       = "./artifacts/models/팀이름/v1"  # ← 실제 경로 맞나?
```

---

## 전체 흐름 요약

```
tactical16 고정
    ↓
기본 보상으로 학습 실행 (Stage 0-1 통과 확인)
    ↓
my_reward.py에서 pursuit_scale / win_reward 조절
    ↓
run_local_dogfight.py로 RL vs BT 테스트
    ↓
my_submission.py에 observation_mode / ACTION_REPEAT / 번들 경로 동기화
    ↓
제출
```
