## 케이스 1 — observation_mode 불일치

**증상**: 로컬 테스트에서는 잘 싸우는데 제출 후 에이전트가 멍하니 직진하거나 즉시 추락

**원인**: `my_train.py`와 `my_submission.py`의 `observation_mode`가 다름

```python
# my_train.py에서 학습
ENV_CONFIG = {"observation_mode": "tactical16"}

# my_submission.py에서 실수로 다르게 설정
OBSERVATION_MODE = "classic12"   # ← 이게 원인
```

훈련된 정책은 16차원 입력을 기대하는데, 제출 시 12차원이 들어오면 완전히 다른 행동 출력.

**해결**: 세 곳 모두 동일하게 설정
```
my_train.py → run_local_dogfight.py --observation-mode → my_submission.py
```

## 케이스 2 — ACTION_REPEAT 불일치

**증상**: 에이전트가 과도하게 흔들리거나, 반대로 너무 느리게 반응함. 로컬과 제출 결과가 다름

**원인**: 훈련 시 `step_ratio=6`이었는데 제출 시 `ACTION_REPEAT=3`으로 설정 (또는 반대)

```python
# 훈련: 매 6 물리 스텝마다 RL 행동 1회
TRAINING_CONFIG = {"step_ratio": 6}

# 제출: 매 3 물리 스텝마다 RL 행동 1회 → 실질 제어 주기가 2배 빨라짐
ACTION_REPEAT = 3   # ← 불일치
```

**해결**: `ACTION_REPEAT = step_ratio` 로 동일하게 맞출 것 (기본 권장값: 6)

## 케이스 3 — throttle 해석 실수

**증상**: 에이전트가 계속 속도가 올라가거나 실속해서 추락함

**원인**: RL 출력 throttle은 [-1, 1]이지만 JSBSim에는 [0, 1]로 변환 후 전달됨

```python
# 내부 변환 (자동 처리됨)
throttle_jsb = (throttle_rl + 1) / 2   # [-1,1] → [0,1]
```

보상 함수나 커스텀 관측에서 throttle 값을 직접 쓸 때 변환 없이 raw RL 값을 쓰면 오계산 발생.

**해결**: `state[StateIndex.THROTTLE_CMD]`는 이미 [0, 1] 변환된 값. RL action[3]과 혼동하지 말 것. 실속 방지를 위해 throttle 중립값은 0.5 (RL 출력 기준 0.0)

## 케이스 4 — pursuit reward만 높고 WEZ에 못 들어가는 경우

**증상**: 보상 로그에서 `pursuit` 컴포넌트는 올라가는데 `damage`가 계속 0. 승률이 오르지 않음

**원인**: Pursuit Shaping은 ATA ≤ 30° 조건, WEZ는 ATA ≤ 1° 조건. 29° 갭이 있어서 에이전트가 WEZ 직전에서 멈추는 로컬 최적에 빠짐

```
Pursuit Shaping 활성 구간:  ATA ≤ 30°  (보상 있음)
WEZ 데미지 발생 구간:       ATA ≤  1°  (실제 피해)
```

**해결**:
- `pursuit_scale` 을 줄이고 `damage_scale` 을 키워 WEZ 진입을 직접 보상
- 또는 ATA < 5° 구간에 추가 shaping 보상 추가

## 케이스 5 — Stage 진급 조건을 못 맞추는 경우

**증상**: 커리큘럼 훈련이 Stage 0이나 1에서 수십 iteration 동안 멈춤

**원인**: 진급 조건 미달. Stage별 조건:

| Stage | 조건 |
|-------|------|
| 0 | crash_rate < 20% |
| 1 | ep_min_distance < 2000m AND crash_rate < 15% |
| 2 | win_rate ≥ 10% OR ep_wez_steps ≥ 10 |

Stage 0에서 멈춘다면 에이전트가 계속 추락하는 것 → 고도 패널티 강화  
Stage 1에서 멈춘다면 적기에 가까이 가지 않는 것 → pursuit_scale 증가

**해결**: `info["outcome"]`과 `info["ep_min_distance"]` 로그로 원인 확인 후 보상 조정

## 케이스 6 — my_reward.py 반환 형식 오류

**증상**: 훈련 시작 즉시 `TypeError` 또는 `ValueError` crash

**원인**: 보상 함수 반환 형식이 틀림

```python
# 잘못된 예
def compute_reward(...):
    return 1.0          # ← float만 반환, dict 없음

# 올바른 형식
def compute_reward(...):
    reward = 1.0
    components = {"survival": 0.01, "pursuit": 0.5}
    return reward, components   # ← (float, dict) 반드시
```

**해결**: 항상 `return total_reward, components_dict` 형태로 반환

## 케이스 7 — 번들 경로 오류

**증상**: `run_local_dogfight.py` 실행 시 `FileNotFoundError` 또는 정책 로드 실패

**원인**: `--ownship-bundle-dir` 경로 아래에 `metadata.json`이나 `policy_weights.pkl`이 없음

```
# 올바른 번들 구조
artifacts/models/팀이름/v1/
├── metadata.json       ← 없으면 오류
└── policy_weights.pkl  ← 없으면 오류
```

**해결**: 훈련 완료 후 해당 폴더에 두 파일이 모두 있는지 확인. `my_train.py`의 `output_name`/`output_tag` 설정이 경로와 일치하는지 확인
